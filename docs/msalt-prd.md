# msalt-nanobot PRD

> **문서 성격**: 사후 정리 PRD. 이미 구현이 완료된 프로젝트의 가치 제안과 의사결정을 회고적으로 기록한다. 외부 공유와 인수인계 양쪽에 사용한다.
>
> **최종 업데이트**: 2026-04-14

---

## 1. 개요

**msalt-nanobot**은 [nanobot](https://github.com/nanobot-ai/nanobot) 프레임워크를 포크해 라즈베리파이 3B+에서 24시간 구동되는 **1인용 개인 AI 비서**다. 텔레그램 채팅을 인터페이스로 (1) 한국·미국 경제 뉴스 자동 브리핑과 (2) 수면·할일·생활 습관 기록을 제공한다.

## 2. 배경과 문제

개인 사용자가 일상에서 마주하는 두 가지 비효율을 해결하기 위해 시작했다.

**경제 정보 수집의 분산.** 평소 챙겨 보는 정보 소스가 한경·매경·조선비즈·Reuters·CNBC RSS와 삼프로TV·슈카월드 유튜브로 흩어져 있다. 매일 아침과 저녁에 사이트와 앱을 돌아다니며 헤드라인을 읽는 데 30분 이상 소요되고, 정작 중요한 변화는 놓치기 쉽다.

**생활 기록의 마찰.** 수면 시간, 할일, 운동·식단을 기록하려면 각각 다른 앱을 열어야 한다. 텔레그램 채팅처럼 자연어로 "어제 11시에 자서 7시에 일어났어"라고 던지면 자동으로 파싱·분류·집계되는 도구가 필요했다.

**기존 솔루션의 한계.** 상용 비서(Notion AI, ChatGPT 웹)는 한국 RSS·유튜브 통합이 약하고, 자체 호스팅이 안 되며, 항상 켜놓고 푸시받기에는 비용이 든다. 라즈베리파이 + OpenAI API 조합이면 월 1~2달러로 동일 기능을 구현할 수 있다고 판단했다.

## 3. 타겟 사용자

**1인 사용자 (msalt 본인)**

| 항목 | 내용 |
|------|------|
| 역할 | 경제·금융에 관심 있는 개인 투자자 |
| 환경 | 한국 (Asia/Seoul), 텔레그램 주 사용 |
| 디바이스 | 텔레그램 모바일 앱 (입력), 라즈베리파이 (백엔드) |
| 기술 수준 | 직접 코드 수정·배포 가능한 개발자 |
| 핵심 니즈 | 짧고 정확한 정보, 마찰 없는 기록, 24시간 무중단 |

다중 사용자 SaaS가 아니다. 동일 코드를 다른 사람이 쓰려면 본인 RPi에 직접 배포하고 SOUL.md/USER.md를 본인 취향에 맞게 수정해야 한다.

## 4. 핵심 기능

### 4.1 경제 뉴스 브리핑

**무엇을.** RSS와 YouTube에서 경제 뉴스를 수집해 GPT로 요약하고 텔레그램으로 푸시한다. 사용자가 직접 묻기도 한다.

- **자동 푸시**: 평일 아침 07:00 (국내장 시작 전), 저녁 19:00 (미국장 시작 전) 하루 2회
- **소스**: 한경·매경·조선비즈 RSS / Reuters·CNBC RSS / 삼프로TV·슈카월드 YouTube
- **출력 형식**: 국내 → 해외 → 유튜브 순서, 카테고리별 3~5개 헤드라인 + 한 줄 요약
- **대화형**: "삼성전자 관련 뉴스 찾아줘", "이번 주 주요 이슈 정리해줘" 같은 자유 질의 지원

**왜.** 분산된 소스를 매번 직접 도는 시간을 0으로 만든다. 출퇴근 직전·직후 시간에 자동으로 손에 쥐여진다.

**성공 기준.**
- 30일 연속 브리핑 누락 0회
- 브리핑 1회에 헤드라인 10~20개, 읽는 시간 3분 이내
- 동일 기사가 RSS와 유튜브에서 중복 등장하면 URL 기반으로 dedup

### 4.2 생활 습관 관리

**무엇을.** 자연어 입력을 자동 분류해 SQLite에 누적하고, 주간 리포트로 패턴을 보여준다.

- **수면 기록**: "어제 23시에 자서 7시에 일어났어" → 취침/기상/수면시간 파싱 → 주간·월간 평균
- **할일 관리**: 추가, 완료 체크, 기한별 조회. 별도 앱 없이 텔레그램에서 완결
- **자유 기록**: "5km 달림", "커피 3잔", "기분 별로" 같은 자유 텍스트 → 키워드 기반 자동 분류 (운동/식단/건강/감정/기타)
- **주간 리포트**: 일요일 밤 자동 생성. 수면 평균, 할일 완료율, 카테고리별 기록 횟수

**왜.** 기록하는 행위 자체의 마찰을 텔레그램 채팅 한 줄로 줄인다. 분류·집계는 봇이 알아서 한다.

**성공 기준.**
- 자유 텍스트 분류 정확도 80% 이상 (키워드 매칭 기준)
- 주간 리포트가 사용자의 행동 패턴에 대한 비판이 아닌 사실 요약으로 구성
- "어제 잠 몇 시간 잤지?" 같은 즉석 조회가 1초 이내 응답

### 4.3 인프라

**무엇을.** 라즈베리파이 3B+에서 systemd 서비스로 자동 시작·재시작되며 항상 켜진 상태를 유지한다.

- 라즈베리파이 3B+ (1GB RAM) 단일 노드
- systemd 서비스 (`msalt-nanobot.service`) 부팅 시 자동 시작
- swap 1GB로 OOM 방지
- nanobot의 cron 스케줄링으로 정기 작업 트리거
- 모든 데이터(SQLite + 메모리 파일)는 `~/.nanobot/workspace/` 단일 경로에 집중

**왜.** 클라우드 의존 없이 본인 집에서 24시간 구동. 월 비용은 OpenAI API 호출비(약 1~2달러)와 전기료뿐.

**성공 기준.**
- 부팅 후 60초 이내 봇 응답 가능
- 30일 연속 무중단 (systemd가 크래시 시 자동 재시작)
- RAM 사용량 500MB 이하 유지

## 5. 비목표 (Non-goals)

명시적으로 **하지 않을** 것들. 이 경계를 흐리면 RPi 자원 한계와 1인 사용자 가정이 무너진다.

- **다중 사용자 지원 안 함.** Telegram allowlist에 단일 user ID만 허용. 인증·권한 시스템 없음.
- **투자 권유·매수매도 추천 안 함.** 뉴스는 사실 전달만. 의견은 사용자 본인 판단.
- **모바일/웹 앱 없음.** 모든 인터페이스는 텔레그램 채팅.
- **음성 입출력 없음.** 텍스트 전용.
- **클라우드 배포 안 함.** 로컬 단일 노드. 다른 사람이 쓰려면 본인 RPi에 직접 배포.
- **실시간 시세·체결 데이터 없음.** 뉴스만 다룬다. 호가·차트는 별도 도구.
- **외부 푸시 광고·뉴스레터 발송 안 함.** 사용자가 직접 보거나 자동 브리핑만.

## 6. 성공 지표

정량적 KPI보다 **정성적 운영 지표**로 본다. 1인 도구이므로 사용량·전환율 같은 SaaS 지표는 무의미하다.

| 지표 | 목표 | 측정 방법 |
|------|------|-----------|
| 브리핑 누락 | 30일 연속 0회 | systemd 로그 + 텔레그램 수신 확인 |
| 응답 지연 | 사용자 메시지 → 응답 5초 이내 | 텔레그램 타임스탬프 |
| 자유 기록 분류 정확도 | 80%+ | 주간 샘플 수동 검토 |
| RAM 사용량 | 500MB 이하 | `free -h` 모니터링 |
| 월 OpenAI 비용 | $3 이하 | OpenAI 대시보드 |
| 사용자 만족 | "매일 열어본다"는 감각 유지 | 본인 주관 평가 |

## 7. 향후 확장 가능성 (참고)

현재 범위 밖이지만, 동일 아키텍처 위에서 자연스럽게 추가 가능한 영역:

- **소스 확장**: X(Twitter), Reddit, 블로그·커뮤니티 크롤링
- **UI 개선**: 텔레그램 인라인 버튼·메뉴
- **시각화**: 주간 리포트에 matplotlib 차트 이미지 첨부
- **음성**: 텔레그램 음성 메시지 → Whisper STT
- **LLM 옵션**: 로컬 LLM(예: llama.cpp) 백엔드 추가로 API 비용 0화

이들은 모두 우선순위 낮음. 현재 구현이 안정적으로 자리잡은 후 검토.

## 8. 주요 의사결정 (ADR 요약)

회고용 핵심 결정 기록. "왜 다른 길로 가지 않았는지" 6개월 뒤 본인이 봐도 이해 가능하도록.

| # | 결정 | 대안 | 선택 이유 |
|---|------|------|-----------|
| 1 | nanobot **fork + upstream sync** | 0) nanobot 미사용, 1) submodule, 2) PR로 본체 기여 | nanobot의 cron·메모리·dream 시스템을 그대로 활용하면서 msalt 전용 기능을 격리. upstream 업데이트는 주기적으로 머지. PR로 보내기엔 1인 특화 기능이라 부적합. |
| 2 | **하이브리드 확장** (skills + 전용 Python 모듈) | 0) 스킬만, 1) 전용 모듈만 | 대화형 진입점은 SKILL.md로 nanobot 자연스럽게 연결, 무거운 로직(RSS 파싱, SQLite, 분류)은 `msalt/` 모듈로 분리. 스킬은 얇게, 모듈은 두껍게. |
| 3 | **Raspberry Pi 3B+** | 0) 클라우드 VM, 1) RPi 4/5, 2) 데스크탑 상시 가동 | 이미 보유 중인 디바이스. 월 전기료 < $1. 1GB RAM 한계는 swap + 경량 모델로 해결 가능. |
| 4 | LLM: **OpenAI gpt-5-mini** | 0) gpt-5/4o, 1) Claude Haiku, 2) 로컬 llama | 비용·품질·한국어 처리 균형. RPi 로컬 추론은 너무 느림. 향후 로컬 옵션 추가 가능. |
| 5 | DB: **SQLite 단일 파일** | 0) PostgreSQL, 1) JSON 파일, 2) Redis | 1인용·동시 쓰기 없음·백업은 파일 복사. 의존성 0 (Python 표준). |
| 6 | 채널: **Telegram 단독** | 0) Slack, 1) Matrix, 2) 자체 웹 UI | 본인이 항상 켜놓는 메신저. 봇 API가 무료·간단. nanobot이 이미 지원. |
| 7 | 브리핑: **하루 2회 고정 시간** | 0) 매시간, 1) 사용자 트리거만, 2) 변동성 기반 알람 | 출퇴근 직전·직후 루틴에 맞춤. 너무 잦으면 피로, 너무 적으면 놓침. 추가 정보가 필요하면 사용자가 직접 질의. |
| 8 | 분류: **키워드 기반** (LLM 미사용) | 0) GPT 분류 호출, 1) ML 모델 학습 | 자유 기록 1건당 LLM 호출은 비용·지연 측면에서 과함. 한국어 키워드 사전으로 80% 정확도면 충분. 오분류는 사용자가 명시 카테고리 입력으로 정정. |
| 9 | 페르소나: **SOUL.md/USER.md 사전 세팅** | 0) nanobot 기본값 사용 후 dream으로 학습 | 첫 실행부터 한국어 반말·간결 톤·뉴스 사실 전달 원칙이 적용되어야 함. dream 학습은 보완용. |
| 10 | 환경 변수: **단일 .env 파일** | 0) config.json에 직접 입력, 1) systemd EnvironmentFile만 | 모든 비밀값(API 키, Telegram user ID)을 한 곳으로. nanobot config는 `${VAR}` 치환만 사용 → 편집 불필요. |
