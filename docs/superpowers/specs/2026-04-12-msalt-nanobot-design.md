# msalt-nanobot 설계 문서

> nanobot 포크 기반 개인 AI 비서 — 경제 뉴스 + 생활 습관 관리

## 1. 프로젝트 개요

### 목표
nanobot 프레임워크를 기반으로 라즈베리파이에서 구동되는 개인 비서를 구축한다.
텔레그램을 통해 경제 뉴스 브리핑과 생활 습관 관리 기능을 제공한다.

### 핵심 기능
1. **경제 뉴스 비서** — 한/미 뉴스, 유튜브 등 고정 소스에서 콘텐츠를 수집하고 요약 브리핑 제공
2. **생활 습관 관리** — 수면 패턴, 할일 관리, 자유 텍스트 기록의 자동 분류/통계/분석
3. **라즈베리파이 + 텔레그램** — Pi 3B+에서 구동, 텔레그램으로 푸시+풀 방식 인터페이스

### 요구사항 요약

| 항목 | 내용 |
|------|------|
| 실행 환경 | Raspberry Pi 3B+ (1GB RAM, ARMv7) |
| LLM 프로바이더 | OpenAI GPT (gpt-5-mini 기본) |
| 인터페이스 | 텔레그램 (자동 푸시 + 대화형 요청) |
| 브리핑 주기 | 하루 2회 (아침 07:00, 저녁 19:00) |
| 코드 구조 | nanobot 포크 유지 + upstream 동기화 |
| 확장 방식 | 하이브리드 — nanobot 스킬/config 활용 + msalt/ 전용 Python 모듈 |
| 개발 순서 | Phase 1 인프라 → Phase 2 경제 뉴스 → Phase 3 생활 습관 |

### 핵심 설계 원칙
- **nanobot 레이어**: 텔레그램 채널, 에이전트 루프, 메모리, 스킬, 크론 — 기존 그대로 활용
- **msalt/ 레이어**: 뉴스 수집기, 생활 습관 트래커 — 별도 디렉토리에 격리된 Python 모듈
- **연결 방식**: msalt 모듈을 nanobot 스킬/도구에서 호출하거나, 크론으로 직접 실행
- **upstream 동기화**: msalt/ 디렉토리에 전용 코드를 격리하여 nanobot 업데이트와 충돌 최소화

## 3. Phase 1 — 인프라 (텔레그램 + 라즈베리파이)

### 목표
nanobot이 라즈베리파이 3B+에서 텔레그램으로 기본 대화가 가능한 상태

### 3.1 config 설정

`~/.nanobot/config.json` 주요 설정:
- **프로바이더**: OpenAI GPT (gpt-5-mini 기본 — 비용/속도 균형)
- **채널**: 텔레그램 활성화, 봇 토큰, allowFrom(본인 계정만)
- **경량화**: maxTokens 절약, 불필요한 채널/도구 비활성화

### 3.2 msalt/ 디렉토리 초기화

```
msalt/
├── __init__.py
├── news/           # Phase 2에서 구현
│   └── __init__.py
├── lifestyle/      # Phase 3에서 구현
│   └── __init__.py
└── config.py       # msalt 전용 설정 (소스 목록, 스케줄 등)
```

### 3.3 라즈베리파이 배포

- systemd 서비스로 `nanobot gateway` 자동 시작
- Python 3.11+ 환경 구성
- swap 1GB 설정 (메모리 부족 대비)

### 3.4 리소스 제약 대응 (Pi 3B+ 1GB RAM)

- LLM은 전부 외부 API 호출 (로컬 모델 없음)
- 동시 세션 1개 제한
- 웹 패치 시 readability 파싱으로 HTML 경량화
- dream 주기를 길게 설정 (6~12시간)
  - dream: nanobot의 메모리 정리 기능. 과거 대화를 주기적으로 요약하여 SOUL.md, USER.md, MEMORY.md에 반영. LLM API를 호출하므로 Pi에서는 빈도를 줄여 부하/비용 절약

## 4. Phase 2 — 경제 뉴스 비서

### 목표
고정된 소스에서 뉴스를 자동 수집하고, 하루 2회 텔레그램으로 요약 브리핑 제공

### 4.1 수집 모듈 (`msalt/news/`)

```
msalt/news/
├── __init__.py
├── collector.py      # 수집 오케스트레이터
├── sources/
│   ├── __init__.py
│   ├── rss.py        # RSS 피드 파서
│   ├── youtube.py    # YouTube Data API
│   └── web.py        # 웹 스크래핑 (RSS 없는 소스용)
├── storage.py        # SQLite 저장
└── sources.json      # 소스 목록 설정
```

### 4.2 초기 수집 소스

| 카테고리 | 소스 | 수집 방식 |
|---------|------|----------|
| 한국 뉴스 | 한경, 매경, 조선비즈 | RSS |
| 미국 뉴스 | Reuters, CNBC | RSS |
| 유튜브 | 삼프로TV, 슈카월드 등 | YouTube Data API |

- 유튜브 영상 요약: YouTube Data API로 자막(transcript)을 가져와 GPT로 요약. 자막이 없는 영상은 제목+설명만 요약
- `sources.json`에서 소스 추가/제거 관리
- 추후 X, Reddit은 소스 모듈만 추가하면 확장 가능

### 4.3 요약 및 브리핑

- 수집된 기사를 GPT에게 전달하여 요약 생성
- 중복 제거: 제목/URL 기반 dedup
- 브리핑 포맷:
  ```
  아침 경제 브리핑 (2026-04-12)

  [국내]
  1. [제목] — 요약 2-3줄
     원문: https://...

  [해외]
  1. [제목] — 요약 2-3줄
     원문: https://...

  [유튜브]
  1. [채널명] 제목 — 핵심 내용 요약
     링크: https://...
  ```

### 4.4 스케줄링

nanobot 크론 시스템 활용:
- 뉴스 수집: 06:30, 18:30
- 아침 브리핑: 07:00
- 저녁 브리핑: 19:00

### 4.5 대화형 기능

텔레그램에서 직접 요청 가능 (nanobot 스킬로 구현):
- "오늘 뉴스 요약해줘" → 최신 브리핑 재전송
- "삼성전자 관련 뉴스" → 키워드 필터링
- "이번 주 뉴스 정리" → 주간 요약

## 5. Phase 3 — 생활 습관 관리

### 목표
텔레그램 대화로 생활 데이터를 기록하고, 패턴 분석 및 통계를 제공

### 5.1 모듈 구조 (`msalt/lifestyle/`)

```
msalt/lifestyle/
├── __init__.py
├── tracker.py        # 기록 처리 오케스트레이터
├── categories/
│   ├── __init__.py
│   ├── sleep.py      # 수면 기록 파서/분석
│   └── todo.py       # 할일 관리
├── classifier.py     # 자유 텍스트 → 카테고리 자동 분류 (GPT)
├── analyzer.py       # 통계/패턴 분석
└── storage.py        # SQLite 저장
```

### 5.2 수면 기록

- **입력**: 자연어 ("어젯밤 11시에 자서 6시에 일어남", "오늘 7시간 잤어")
- **파싱**: GPT가 자연어 → 구조화 데이터 추출 (취침시간, 기상시간, 수면시간)
- **분석**: 주간/월간 평균, 취침시간 추이, 규칙성 점수

### 5.3 할일 관리

- **입력**: "내일 오후 3시 치과 예약", "장보기 추가해줘", "치과 완료"
- **기능**: 추가, 완료 표시, 목록 조회, 리마인더
- **리마인더**: 크론으로 예정 시간 전 텔레그램 알림
- **연동**: 매일 저녁 브리핑에 미완료 할일 포함

### 5.4 자유 텍스트 기록 + 자동 분류

- **입력**: "오늘 5km 달림", "커피 3잔 마심", "두통약 먹음", "기분 좋음"
- **분류**: GPT가 카테고리 자동 분류 (운동, 식단, 건강, 감정 등)
- **저장**: 구조화하여 SQLite에 저장
- **유연한 스키마**: 새로운 카테고리는 자동 생성

### 5.5 통계 및 분석

대화형 요청:
- "이번 주 수면 통계" → 평균, 최소/최대, 추이
- "지난 달 운동 기록" → 횟수, 종류별 분류
- "최근 패턴 분석해줘" → GPT가 전체 데이터 기반 인사이트 제공

주간 리포트: 일요일 저녁 브리핑에 주간 생활 요약 자동 포함

### 5.6 데이터 스키마 (SQLite)

```sql
-- 수면 기록
CREATE TABLE sleep_log (
    date TEXT PRIMARY KEY,
    bedtime TEXT,
    wakeup TEXT,
    duration_min INTEGER,
    quality TEXT
);

-- 할일
CREATE TABLE todos (
    id INTEGER PRIMARY KEY,
    content TEXT,
    due_at TEXT,
    completed_at TEXT,
    status TEXT  -- pending, done, cancelled
);

-- 자유 기록
CREATE TABLE life_log (
    id INTEGER PRIMARY KEY,
    timestamp TEXT,
    raw_text TEXT,
    category TEXT,
    parsed_data TEXT  -- JSON
);
```
